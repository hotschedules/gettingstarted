REST API

The REST API lets you interact with RBC Cloud from anything that can send an HTTP request. There are many things you can do with the REST API. For example:

  • A mobile website can access RBC data from Javascript.
  • A webserver can show data from RBC on a website.
  • You can upload large amounts of data that will later be consumed in a mobile app.
  • You can export all of your data if you no longer want to use RBC.
  • You can download recent data to run your own custom analytics.
  • Applications written in any programming language can interact with data in the RBC Cloud.

Quick Reference

RBC Cloud stores data in the form of Types, which are JSON-like field and value pairs. Types are analogous to structures in programming languages that associate keys with values (e.g. dictionaries, hashes, maps, and associative arrays). RBC Cloud stores all Types in Collections. A Collection is a group of related Types that have a set of shared common indexes. Collections are analogous to a table in relational databases.

All API access is over HTTPS, and accessed via the https://api.rbcplatform.com domain. All API requests must include the tenant name or namespace in the URL before referencing the resource type and any qualifiers.

Request Format

For POST and PUT requests, the request body must be JSON, with the Content-Type header set to application/json.

Authentication is achieved via HTTP headers. In the current version of the API you must authenticate your REST API requests using basic HTTP authentication with Base 64 encoded strings. For example, to retrieve an object you could set the URL using your RBC credentials in the following format:

var auth = "Basic " + new Buffer(userid + ":" + password).toString("base64");

Response Format

The response format for all requests is a JSON object.

Whether a request succeeded is indicated by the HTTP status code. A 2xx status code indicates success, whereas a 4xx status code indicates failure. When a request fails, the response body is still JSON, but always contains the fields code and error which you can inspect to use for debugging.
